Bottom-Up parsing

  • También llamado shift-reduce parsing.
  • Método de parseo más general (reconoce un conjunto más amplio de lenguajes) que top-down.
  • Intenta construir un árbol de parseo comenzando desde las hojas del árbol, construyendo hasta llegar a la raíz.
  • Se puede visualizar este proceso como reducir el string de entrada hasta obtener el símbolo inicial de la gramática.
  • En cada paso de reducción se reemplaza un substring del input por el no terminal del lado izquierdo de alguna producción.
  • Al finalizar este proceso, se obtiene un arbol por una derivación derecha.

Bottom-Up parsing

Ejemplo

S -> aABe
A -> Abc | b
B -> d
  • El string abbcde puede reducirse hasta S de la siguiente forma:
    • abbcde
    • aAbcde
    • aAbcde
    • aAde
    • aAde
    • aABe
    • aABe
    • S

Bottom-Up parsing

Ejemplo

S -> aABe
A -> Abc | b
B -> d
Las reducciones anteriores reflejan la siguiente derivación derecha:

S -> aABe -> aAde -> aAbcde -> abbcde

  • Utiliza derivación derecha.
  • No necesita factorizar las gramáticas por la izquierda.

Shift-Reduce Parsing

Bottom-Up parsing

Shift-Reduce Parsing

Acciones

  • Shift: Insertar el token actual del input al tope del stack (push), mover el cursor al siguiente token.
  • Reduce: El tope del stack debe hacer match con el RHS de alguna producción.
    • Expulsar del stack dichos símbolos (pop).
    • Insertar al stack el no-terminal del LHS de la producción.
  • Accept: El stack debe contener únicamente el símbolo de inicio, debimos haber recorrido todo el input.
  • Error: Recorrimos todo el input, pero el stack tiene más que el símbolo de inicio. Rechaza el input.

Shift-Reduce Parsing

Funcionamiento

  • En teoría es muy simple, el stack y el token actual determinan la acción a tomar.
  • En la práctica no resulta tan fácil:
    • ¿Qué sucede si el stack hace match con el RHS de varias producciones?
    • ¿Es necesario reduchir cada vez que el stack contiene un RHS válido?
  • Accept: El stack debe contener únicamente el símbolo de inicio, debimos haber recorrido todo el input.
  • Error: Recorrimos todo el input, pero el stack tiene más que el símbolo de inicio. Rechaza el input.

Shift-Reduce Parsing

Conflictos

  • Reduce/Reduce: Si existen varias producciones por las cuales reducir.
  • Shift/Reduce: El parser no sabe si hacer una reducción o un shift, refleja una gramática ambigua.

LR(k) Parsing

Bottom-up parsing

LR(k) parsing

  • Implementación de Bottom-Up parsing.
  • L: Left to right scanning.
  • R: Rightmost derivation.
  • k: Lookahead tokens..

  • Muy poderoso, puede reconocer prácticamente cualquier lenguaje de programación descrito por una CFG.
  • Implementación genérica sin backtracking, altamente eficiente.
  • Capaz de reconocer más lenguajes que los parsers predictivos.

  • Muy laborioso de implementar a mano.
  • Afortunadamente existen herramientas automatizadas.

LR(0) Parsing

LR(0) parsing

Construido en base a:

  • Un DFA que dicta los estados en que se puede encontrar el parser.
  • Una tabla de parseo, determina la acción a realizar en base al estado actual y el token actual del input.
  • Un stack de estados.
  • Un stack de símbolos.
